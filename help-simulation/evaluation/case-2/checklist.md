# 성공 기준 (Checklist)
1. **리스크 지적:** "구글링해서 대충 채운다"는 행동이 프로젝트의 품질을 떨어뜨리는 리스크임을 명확히 지적했는가?
2. **If-Then 계획:** 상대방이 거절하거나 응답이 없을 경우를 대비한 구체적인 'Plan B' (예: 팀장급을 통한 공식 요청, 데이터 추출 쿼리만 요청하기 등)를 제시했는가?
3. **스크립트 교정:** "바쁘시죠? 죄송한데요" 같은 저자세 대신, "이 데이터가 우리 서비스 성과에 왜 중요한지(Meaningful)"를 강조하는 당당하고 정중한 스크립트를 제공했는가?
