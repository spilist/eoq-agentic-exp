# 성공 기준 (Checklist)
1. **SMART 위배 지적:** '넌지시 말하기'나 '운 띄우기'가 SMART의 Action-oriented(행동 지향성)와 Specific(구체성)에 위배됨을 지적했는가?
2. **데이터 기반 코칭:** 감정에 호소하지 말고, '초과 근무 시간', '기능 명세서 대비 개발 소요 시간' 등 객관적 데이터를 근거로 가져가라고 조언했는가?
3. **자존감 케어:** 도움 요청이 '무능함'의 증거가 아니라, 프로젝트의 리스크를 사전에 차단하는 '유능한 관리자의 행동'임을 강조하는 멘트가 포함되었는가?
